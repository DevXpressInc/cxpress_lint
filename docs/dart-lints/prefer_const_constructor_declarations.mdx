# prefer_const_constructor_declarations

| Severity | Quick Fix | Options |
|:--------:|:---------:|:-------:|
|   Info   |    ✅     |   ❌    |

## Details

**DO** declare constructors as const when possible.

```dart title="Bad"
class Point {
  final double x;
  final double y;

  Point(this.x, this.y);

  Point.origin()
      : x = 0,
        y = 0;
}
```

```dart title="Good"
class Point {
  final double x;
  final double y;

  const Point(this.x, this.y);

  const Point.origin()
      : x = 0,
        y = 0;
}
```

## Usage

To enable the `prefer_const_constructor_declarations` rule, add `prefer_const_constructor_declarations` under custom_lint > rules in your `analysis_options.yaml` file:

```yaml
custom_lint:
  rules:
    - prefer_const_constructor_declarations
```
